# Условие задания
### Найдите баги на приложенном скриншоте, приоритезируйте их и перечислите списком. Приоритеты: P0 — критический, P1 — высокий, P2 — средний, P3 — низкий.
#### Пример: P0 — не открывается объявление из выдачи.

### Описание багов:
#### Формат:
N) Название
1. Приоритет
2. Объяснение приоритета
3. Шаги воспроизведения
4. Ожидаемое поведение
5. Рекомендации

#### Найденные баги:
1) Неправильная иерархия разделов в поиске
1. P2 (Medium)
2. Баг не блокирует функциональность: пользователь может найти объявления, переход по ссылкам хлебных крошек (если они кликабельны) ведет в соответствующие разделы. Однако нарушение иерархии ухудшает UX и может запутать пользователя. Это визуальный баг среднего уровня, который влияет на восприятие качества продукта, но не требует немедленного исправления
3. 1. Открыть главную страницу сайта. 2. Перейти в категорию «Животные». 3. Выбрать подкатегорию «Кошки». 4. Выбрать породу «Персидская». 5. Обратить внимание на разделы вверху страницы
4. В навигации : Главная/Животные/Кошки/Персидская(или Порода Персидская)
5. Исправить логику формирования разделов в соответствии с иерархией категорий. Проверить корректность отображения для всех вложенных категорий

3. 1. Открыть главную страницу сайта. 2. Посмотреть на верхнюю панель навигации
4. В навигации (верхняя плашка) : "Карьера в Авито"
5. Исправить опечатку в текстовом ресурсе (строке) интерфейса. Проверить наличие аналогичных ошибок в других версиях

3. 1. Открыть главную страницу сайта. 2. Выполнить поиск любого товара. 3. Обратить внимание на текст рядом с чекбоксом или кнопкой сохранения поиска
4. В поиске : "Уведомлять"
5. Исправить опечатку, проверить аналогичную ошибку на других страницах

5) Ложное сообщение об ошибке отображается при корректной выдаче
1. P2 (Medium)
2. Сообщение об ошибке появляется, хотя объявления успешно загружены и отображаются. Это может дезориентировать пользователя и вызвать недоверие. Функционально поиск работает, поэтому приоритет средний. Требования к шрифту и выравниванию являются отдельным пожеланием (UI), не критичным.
3. 1. Открыть главную страницу сайта. 2. Выполнить поиск по любому запросу. 3. Обратить внимание на наличие сообщения об ошибке над или под выдачей.
4. При успешной загрузке объявлений сообщения об ошибке не отображаются. При реальной ошибке загрузки сообщение должно быть заметным (крупный шрифт, центрирование).
5. Исправить логику отображения ошибки: сообщение должно показываться только при фактической ошибке загрузки данных.

6) В выдаче по запросу «Персидская» отображается объявление с породой «Экзотический короткошерстный»
1. P3 (Low)
2. Если поиск осуществляется по точному совпадению породы, это баг релевантности. Однако экзотическая короткошерстная порода является родственной персидской, и в некоторых системах поиска может считаться релевантным результатом. Необходимо уточнить требования. Баг не блокирует функциональность и не влияет на большинство пользователей.
3. 1. Открыть страницу поиска. 2. В поле «Порода» выбрать или ввести «Персидская». 3. Просмотреть выдачу.
4. Показ объявлений четко соответствующих выбранной фильтрации
5. Уточнить требования к поиску по породам: должен ли поиск выдавать только точное совпадение или включать родственные породы. Привести функционал в соответствие с требованиями.

8) Фильтр по полу не применяется: при выборе «Мальчик» отображаются объявления с «Девочка»
1. P1 (High)
2. Фильтрация по полу является ключевым параметром поиска для пользователей, ищущих животных. Некорректная работа фильтра приводит к показу нерелевантных объявлений, что вынуждает пользователя вручную просматривать результаты. Баг напрямую влияет на пользовательский опыт.
3. 1. Открыть страницу поиска в категории «Кошки». 2. В фильтре «Пол» выбрать значение «Мальчик». 3. Нажать «Применить» / «Показать». 4. Просмотреть выдачу.
4. В результатах корректное отображение с фильтрацией
5. Проверить корректность передачи параметра пола в запросе к API. Проверить обработку параметра на бэкенде. Проверить соответствие данных в объявлениях.

3. 1. Открыть страницу поиска. 2. В фильтре «Цена» установить диапазон (например, от 1000 до 30000). 3. Нажать «Применить» / «Показать». 4. Просмотреть выдачу.
4. В результатах корректное отображение с фильтрацией
5. Проверить корректность передачи параметров цены в запросе к API. Проверить обработку диапазона цен на бэкенде. Проверить, корректно ли сравниваются цена в объявлении и условия фильтра

Баг не блокирует функциональность, но негативно влияет на восприятие геосервисов платформы.
3. 1. Установить город поиска — Москва. 2. Выполнить поиск в любой категории. 3. Обратить внимание на наличие плашки с предложением «Сначала из [город]».
